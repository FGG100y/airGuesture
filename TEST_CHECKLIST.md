# AirGesture 功能测试清单

## 测试准备
- [ ] 设备已连接并启用USB调试
- [ ] 已安装最新版本APK
- [ ] 授予所有必要权限（相机、悬浮窗、无障碍）

## 基础功能测试

### 1. 权限和服务启动
- [ ] 相机权限可以正常请求和授予
- [ ] 悬浮窗权限可以正常设置
- [ ] 无障碍服务可以正常启用
- [ ] 服务启动后显示前台通知
- [ ] 悬浮窗图标正常显示

### 2. 手势识别测试

#### 滑动手势
- [ ] **左滑** → 上一页（在浏览器或PDF阅读器中测试）
- [ ] **右滑** → 下一页（在浏览器或PDF阅读器中测试）
- [ ] **上滑** → 向上滚动（在任意可滚动界面测试）
- [ ] **下滑** → 向下滚动（在任意可滚动界面测试）

#### 静态手势
- [ ] **OK手势** → 截图功能
- [ ] **握拳** → 停止滚动
- [ ] **张开手掌** → 播放/暂停（在媒体应用中测试）
- [ ] **剪刀手** → 返回主页

#### 特殊手势
- [ ] **顺时针画圈** → 打开最近任务
- [ ] **L形手势** → 返回上一界面

### 3. 配置功能测试
- [ ] 打开手势配置界面
- [ ] 修改手势映射并保存
- [ ] 验证修改后的映射生效
- [ ] 恢复默认设置功能正常

### 4. 视觉反馈测试
- [ ] 手势方向指示器正常旋转
- [ ] 手势识别后有相应的视觉反馈
- [ ] 悬浮窗不影响其他应用操作

## 稳定性测试

### 5. 异常情况处理
- [ ] 手快速移动时不会误触发
- [ ] 无手势时保持稳定不触发
- [ ] 服务异常终止后可以重新启动
- [ ] 权限被撤销后有适当提示

### 6. 性能测试
- [ ] CPU使用率保持在合理范围
- [ ] 内存占用稳定，无内存泄漏
- [ ] 电池消耗在可接受范围内
- [ ] 长时间运行稳定性

## 兼容性测试

### 7. 不同应用测试
- [ ] 浏览器（Chrome/Firefox）
- [ ] PDF阅读器
- [ ] 电子书应用
- [ ] 演示文稿应用
- [ ] 社交媒体应用（微信、抖音等）

### 8. 系统兼容性
- [ ] Android 7.0（API 24）最低版本
- [ ] Android 13（API 33）权限兼容
- [ ] Android 14（API 34）正常工作

## 已知问题

### 需要注意的限制
1. 音量控制功能在某些设备上可能无法正常工作
2. 媒体控制功能需要媒体应用在前台
3. 截图功能需要Android 9（API 28）或更高版本
