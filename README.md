# AirGesture - 手势控制应用

## 项目简介
AirGesture 是一款基于 MediaPipe 手部追踪技术的 Android 应用，通过摄像头识别手势来控制设备，实现翻页、滚动、截图等功能，无需触摸屏幕即可操作。

## 主要功能

### 支持的手势和动作
- **左滑** → 上一页
- **右滑** → 下一页  
- **上滑** → 向上滚动
- **下滑** → 向下滚动
- **OK手势** → 截图
- **握拳** → 停止滚动
- **张开手掌** → 播放/暂停
- **剪刀手** → 返回主页
- **顺时针画圈** → 最近任务
- **L形手势** → 返回

## 系统要求
- Android 7.0 (API 24) 或更高版本
- 支持前置摄像头的设备
- 建议 2GB 以上 RAM

## 安装步骤

### 方式一：使用预编译 APK
1. 下载 `app-debug.apk` 文件
2. 在设备上启用"允许安装未知来源应用"
3. 安装 APK 文件

### 方式二：从源码编译
```bash
# 克隆项目
git clone [项目地址]

# 进入项目目录
cd airGesture

# 编译调试版
./gradlew assembleDebug

# 安装到设备
./gradlew installDebug
```

## 使用指南

### 初始设置
1. **授予权限**
   - 相机权限：用于手势识别
   - 悬浮窗权限：显示手势指示器
   - 无障碍服务：执行系统操作

2. **启动服务**
   - 所有权限授予后，点击"启动服务"
   - 服务会在后台运行并显示通知

3. **配置手势**
   - 点击"手势配置"自定义手势映射
   - 可根据个人习惯调整手势对应的动作

### 使用技巧
- 保持手部在摄像头视野内
- 手势动作要明确，避免过快
- 良好的光线环境可提高识别准确度
- 建议距离摄像头 30-60cm

## 技术架构

### 核心技术
- **MediaPipe** - Google 的手部追踪框架
- **Kotlin** - 主要开发语言
- **C++** - 手势分析算法
- **AccessibilityService** - 系统操作执行

### 项目结构
```
app/
├── src/main/java/com/device/airgesture/
│   ├── service/          # 核心服务
│   ├── action/           # 手势和动作定义
│   ├── analyzer/         # 手势分析器
│   └── utils/            # 工具类
├── src/main/cpp/         # C++ 原生代码
└── src/main/res/         # 资源文件
```

## 开发指南

### 添加新手势
1. 在 `Gesture.kt` 中添加枚举
2. 在 `GestureAnalyzer.cpp` 中实现识别逻辑
3. 在 `GestureActionManager.kt` 中配置映射

### 添加新动作
1. 在 `Action.kt` 中添加枚举
2. 在 `KeyActionExecutor.kt` 中实现执行逻辑
3. 更新配置界面

详细开发文档请参考 [AGENTS.md](AGENTS.md)

## 已知限制
- 音量控制在某些设备上可能不可用
- 截图功能需要 Android 9 (API 28) 或更高版本
- 暗光环境下识别准确度会降低

## 故障排除

### 服务无法启动
- 确认所有权限已授予
- 重启应用或设备
- 检查是否有其他应用占用摄像头

### 手势识别不准确
- 调整手部与摄像头的距离
- 确保良好的光线条件
- 在配置中调整灵敏度

### 应用崩溃
- 查看日志：`adb logcat | grep AirGesture_`
- 清除应用数据后重试
- 确保设备满足最低系统要求
