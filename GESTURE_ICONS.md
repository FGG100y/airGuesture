# AirGesture 手势图标系统说明

## 图标设计

应用使用简洁的图形来提供手势反馈，每个图标都有特定的含义和颜色：

### 图标列表

| 图标文件 | 颜色 | 含义 | 触发条件 |
|---------|------|------|---------|
| `ic_gesture_idle.xml` | 红色 (#F44336) | 空闲状态 | 没有检测到手势时 |
| `ic_gesture_screenshot.xml` | 绿色 (#4CAF50) | 截图/特殊动作 | OK手势、握拳等静态手势 |
| `ic_gesture_left.xml` | 蓝色 (#2196F3) | 向左滑动 | 手向左滑动时 |
| `ic_gesture_right.xml` | 蓝色 (#2196F3) | 向右滑动 | 手向右滑动时 |
| `ic_gesture_up.xml` | 蓝色 (#2196F3) | 向上滑动 | 手向上滑动时 |
| `ic_gesture_down.xml` | 蓝色 (#2196F3) | 向下滑动 | 手向下滑动时 |

## 动画效果

### 1. 空闲状态
- 显示红色圆圈
- 轻微的缩放动画（0.9x）
- 透明度 70%

### 2. 方向手势（滑动）
- 快速切换到对应箭头图标
- 缩放动画：0.8x → 1.1x → 1.0x
- 透明度变化：0.3 → 1.0
- 持续时间：280ms

### 3. 截图手势（特殊动作）
- 显示绿色实心圆圈
- 脉冲效果：0.7x → 1.2x → 1.0x
- 强调动画，表示动作已触发
- 持续时间：350ms

### 4. 自动恢复
- 手势完成后 800ms 自动恢复到空闲状态
- 平滑过渡，避免突兀

private fun updateGestureIcon(gesture: String?, direction: Int) {
    // 根据手势类型选择图标
    val iconResource = when {
        gesture == "OK_SIGN" || gesture == "FIST" -> R.drawable.ic_gesture_screenshot
        direction == -1 -> R.drawable.ic_gesture_left
        direction == 1 -> R.drawable.ic_gesture_right
        direction == 2 -> R.drawable.ic_gesture_up
        direction == -2 -> R.drawable.ic_gesture_down
        else -> R.drawable.ic_gesture_idle
    }

}

## 用户体验优化

1. **视觉反馈清晰**
   - 红色表示等待状态
   - 蓝色表示移动方向
   - 绿色表示动作触发

2. **动画流畅**
   - 所有切换都有过渡动画
   - 避免视觉跳跃

3. **状态明确**
   - 用户能清楚知道：
     - 系统是否在监听（红色圆圈）
     - 手势被识别的方向（箭头）
     - 特殊动作是否触发（绿色圆圈）
